Permissions Tab — Matrix Style

How should the role × page permission grid look and behave?

A

Simple Toggle Grid

Pages down the left, roles across the top as column headers. Each cell is a toggle switch. Role columns are color-coded. Locked cells (Admin full access, /settings always on) shown as faded with a lock icon. Clean and scannable.

            Admin    Manager   Member   Dev
Dashboard   🔒      ✅       ❌      ✅
Messages    🔒      ✅       ❌      ❌
Tasks       🔒      ✅       ✅      ✅
Members     🔒      ❌       ❌      ❌
Settings    🔒      🔒       🔒      🔒
B

Toggle Grid + Per-Role Quick Actions

Same grid as A, but each role column header has a "Grant All / Revoke All" quick action button. Useful when setting up a new role — one click to start from full access then remove, or start blank and add. Also shows a mini progress bar under each role name showing % access.

            Admin      Manager     Member
            [lock]     [▓▓▓▓░░]   [▓░░░░░]
                    Grant/Revoke   Grant/Revoke
Dashboard   🔒       ✅         ❌
Messages    🔒       ✅         ❌
Tasks       🔒       ✅         ✅
C

Toggle Grid + Per-Page Quick Actions

Same grid but with row-level actions — each page row has a "Grant to All / Revoke from All" button. Useful when you want to quickly open or close a page across all roles at once. Role headers show the progress bar too.

            Admin   Manager   Member   [Row action]
Dashboard   🔒     ✅       ❌      [Grant all]
Messages    🔒     ✅       ❌      [Grant all]
Tasks       🔒     ✅       ✅      [Revoke all]

My recommendation: B — Grant/Revoke All per column is a huge time-saver when creating a new custom role. The progress bar gives instant visual feedback on how permissive each role is.
