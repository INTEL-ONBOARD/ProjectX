Users Tab — Directory + Detail

How should user oversight look? You want role assignment, profile editing, and workload/activity info.

A

Table + Slide-out Profile Drawer

Rich table showing avatar, name, email, role (dropdown), workload bar, task count, and status. Clicking any row slides open a right drawer with full profile — editable name, designation, location, status toggle, task activity feed, and workload breakdown. Same pattern as the current Members page profile panel.

Name      Email         Role      Workload   Tasks
● Alice    alice@co      [Manager▼] ████░     8
● Bob      bob@co        [Member▼]  ██░░░     3
                                              → drawer slides in
B

Card Grid + Expanded Card Detail

Users shown as cards (avatar, name, role badge, task count). Clicking a card expands it in-place (like accordion) to show full editable profile + workload stats. No drawer — everything stays in the grid. More visual but takes more vertical space.

┌──────────┐ ┌──────────┐ ┌──────────┐
│ 👤 Alice │ │ 👤 Bob │ │ 👤 Carol │
│ Manager │ │ Member │ │ Dev │
│ 8 tasks │ │ 3 tasks │ │ 5 tasks │
└──────────┘ └──────────┘ └──────────┘
↕ click to expand inline
C

Table + Inline Expandable Row

Compact table. Clicking a row expands it into a full-width detail panel below the row — shows editable fields, workload chart, assigned tasks list. The table stays visible above, making it easy to compare users or quickly jump to another. Best of both table density and drawer richness.

My recommendation: A — The slide-out drawer gives the most room for rich profile editing and task activity without disrupting the table. It also matches the existing Members page pattern your app already uses, so it feels consistent.
