Organization Screen Layout

Which overall structure feels right for managing roles, permissions, and users?

A

Three-Tab Layout (Evolved Current)

Keep the left sidebar nav but expand it: Overview · Users · Roles & Permissions. The Roles tab becomes a full role builder — create/delete roles, name them, toggle permissions. Users tab gets richer profile info and workload stats.

✓ Familiar structure
✓ Easy to extend
— Role builder in sidebar feels cramped
B

Split-Panel: Users + Roles Side by Side

Left panel: full user directory with search, filters, workload bars, inline role assignment. Right panel: role builder — list of all roles, click a role to expand its permission toggles. Selecting a user in the left highlights who has that role on the right.

✓ See users & roles at once
✓ Fast role assignment
— Less room for user detail
C

Four-Tab Layout with Dedicated Role Builder

Left sidebar expands to four sections: Overview · Users · Roles · Permissions. Users = rich directory with profile drawer, workload & tasks. Roles = create/rename/delete roles, see who's assigned. Permissions = matrix view — roles across columns, pages down rows, toggle cells.

✓ Cleanest separation
✓ Scales to many roles
✓ Matrix is powerful

My recommendation: C — Four tabs gives each concern its own space. The permissions matrix is especially powerful once you have 4-6 custom roles — a grid beats individual cards at that scale.
